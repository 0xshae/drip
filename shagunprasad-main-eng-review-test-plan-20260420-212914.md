## Affected Pages/Routes

- `POST /provision` — create user + spin up BWL container
- `POST /teardown` — scale-to-zero user container
- `POST /restore` — scale-back-up user container
- `POST /webhook/checkout-paid` — top-up webhook handler
- `GET /logs` — live agent reasoning log stream
- `GET /dashboard` — credit balance meter + status badge
- `GET /health` — required by BWL, must return 200

## Key Interactions to Verify

- **Provision flow**: POST /provision → BWL service created → deployment polled to healthy → user_status = active
- **Teardown flow**: Manually decrement SQLite balance to 0 → polling loop detects → PATCH BWL minInstances=0 → user_status = paused → AgentMail warning sent
- **Restore via webhook**: Hit POST /webhook/checkout-paid with valid HMAC + user_id metadata → PATCH BWL minInstances=1 → new deployment → user_status = active → AgentMail "container waking up"
- **Research loop (demo_mode=true)**: POST /research/trigger?demo_mode=true → Exa search → Firecrawl scrape → Claude synthesis → AgentMail digest delivered
- **Budget tradeoff**: Set user balance to 25% of initial → trigger research loop → verify only 2 sources crawled (not 5) → agent log shows "budget tight" decision
- **Agent log stream**: /logs endpoint returns last 50 log lines, HTMX polls every 2s, new entries appear live
- **Checkout top-up**: Create checkout session for user → simulate checkout.session.paid webhook → container restores

## Edge Cases

- BWL JWT token expires during demo — Drip must re-exchange token before each BWL call
- User has $0 balance and triggers research loop — loop should abort, not deduct negative credits
- Webhook HMAC signature mismatch — must return 400, not trigger restore
- Polling loop runs while teardown is in progress — idempotent: if status already = paused, skip
- Two requests to /provision for same user_id — must not create duplicate BWL services
- demo_mode=true: credit drain must hit 20% within 2 minutes (calibrate drain rate)

## Critical Paths

- **The full demo loop**: deposit credits → provision → research runs → low-credit warning → drain to 0 → teardown → webhook top-up → restore → research resumes
- **Agent makes a decision**: balance drops below 30% → research loop detects → crawls 2 sources → logs "budget tight — crawling 2 sources instead of 5" — this line must appear in /logs
- **State survives hibernation**: tear down container → inspect Postgres — topics, digest history still present — restore container — research picks up where it left off

## Pre-Demo Smoke Test Script

Run `python smoke_test.py` before every demo. It should:
1. POST /provision with test user + topic
2. Poll until user_status = active (timeout 5 min)
3. POST /research/trigger?demo_mode=true
4. Wait for AgentMail digest (check inbox)
5. Decrement SQLite balance to 0 manually
6. Wait for teardown (confirm via GET /v1/services/{id}?include=runtime → runningCount=0)
7. POST /webhook/checkout-paid with test payload
8. Poll until user_status = active again
9. Confirm /logs has "budget tight" and "container paused" entries
10. Report PASS/FAIL per step
